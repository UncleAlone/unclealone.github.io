1 ADT and Data Structure

1.1 ADT (Abstract Data Type)

  • about the data it manages and the operations it supports
  • not about how it is implemented
  • It is an interface (like the public methods/attributes of an object)

1.2 Data Structure

  • A chosen implementation strategy for an ADT
  • Decides how to actually store the data and implement the operations

1.3 Purpose of ADTs

  • ADTs handle common patterns fo manipulating data
  • Help us work and communicate at higher levels of abstraction

2 Stack

2.1 Data

  • A collection of items

2.2 Features

  • Last-In-First-Out (or LIFO)

2.3 Operations

  • determine whether the stack is empty (is_empty())
  • add an item (push())
  • remove the most recently-added item (pop())

3 Queue

3.1 Data

  • A collection of items

3.2 Features

  • First-in-First-Out (or FIFO)

3.3 Operations

  • determine whether the stack is empty (is_empty())
  • add an item (enqueue())
  • remove the least recently-added item (dequeue())

4 Exceptions

An exception is an object that indicates an exceptional situation (not necessarily problem or error) and it allows error handling and recovery are isolated from the regular flow

  • An exception gets raised during program execution and transfers control to an exception handler
  • Exceptions must be “caught” by an exception handler to avoid the program crashing

4.1 Examples of Exceptions:

>>> 1 / 0
Traceback (most recent call last):
File "<stdin>", line 1, in <module>
ZeroDivisionError: division by zero

4.2 Raising Exceptions

>>> raise ValueError
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ValueError: invalid

4.3 User-Defined Exceptions

Make new types of exceptions by inherit from Exception: 

class MyError(Exception):
    pass

>>> raise MyError
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
__main__.MyError

4.4 Handling Exceptions

# Put the codes with exception in try block
try:
    f = open('myfile.txt')
    s = f.readline()
    i = int(s)
# If except block mathches exception, this block will handle the exception
except IOError:
    print('Input/Output error')
# There can be more than on except block matches (orders matter and use the first)
except ValueError:
    print('Could not convert data to an integer')
print('continuing')

4.5 Execution with Exceptions

  • If no exception occurs, no except block is executed
  • If an exception occurs and an except blck handles it, execution continues following the enclosing try block
  • If an exception occurs and no except block handles it, the exception propagates up the function call stack until it is handled or terminates the program
